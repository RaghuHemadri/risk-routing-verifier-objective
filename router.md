## Router $r_\psi$: Full Theory

### 1. Role

The router sits between a small language model (SLM, 7–13 B params) and a large teacher LLM. At every agent step $t$, the router outputs a **calibrated probability** $r_\psi(x_t) \in [0,1]$ indicating whether the agent should **fall back to the expensive LLM** instead of using the cheap SLM action. It is the mechanism that turns a fixed SLM+LLM pair into a cost-efficient, robust agent.

---

### 2. Architecture

The router is a **lightweight MLP** (a few thousand parameters):

| Component | Detail |
|---|---|
| **Input** | A 13-dimensional feature vector (see §4 below) |
| **Hidden layers** | Configurable stack (default: two layers of 128 units each) with **GELU** activations, **BatchNorm**, and **Dropout** (default 0.2) |
| **Output head** | Single linear unit → **temperature-scaled sigmoid** → probability in $[0,1]$ |
| **Learnable parameters** | (a) MLP weights/biases, (b) a temperature scalar $T$ for calibration, (c) $\log\lambda$ — the log-Lagrange multiplier for the CVaR constraint |

The forward pass (router.py):

$$p = \sigma\!\left(\frac{\mathrm{MLP}(f)}{T}\right)$$

At **inference**, a hard decision is made by thresholding: $d_t = \mathbb{1}[p > 0.5]$ (threshold configurable).

The Lagrange multiplier is stored in log-space and accessed via `softplus` to guarantee $\lambda \ge 0$:

$$\lambda = \mathrm{softplus}(\log\lambda)$$

---

### 3. Objective Function (Training Loss)

Training follows a **constrained optimization** formulated as a **Lagrangian saddle-point** problem:

$$\min_\psi \;\max_\lambda \;\; \underbrace{\mathbb{E}\bigl[\mathrm{cost}(d_{1:T})\bigr]}_{\text{cost term}} + \;\lambda\!\left(\underbrace{\mathrm{CVaR}_\alpha(1 - S_z)}_{\text{tail failure rate}} - \epsilon\right) + \;\underbrace{w_B \cdot \mathrm{Brier}(p, y)}_{\text{calibration}}$$

It decomposes into four sub-losses:

#### (a) Cost loss

$$\mathcal{L}_{\text{cost}} = \mathbb{E}\bigl[p \cdot c_{\text{LLM}} + (1-p)\cdot c_{\text{SLM}}\bigr]$$

Where $c_{\text{SLM}}=1$ and $c_{\text{LLM}}=50$ (defaults). This penalizes unnecessary LLM calls.

#### (b) Robustness / CVaR constraint

The key innovation. Episodes are grouped by **perturbation seed** $z$. For each seed, the failure rate is computed:

$$F_z = \frac{1}{|B_z|}\sum_{i \in B_z}(1 - S_i)$$

Then $\mathrm{CVaR}_\alpha$ takes the **average of the worst $\alpha$-fraction** of seeds:

$$\mathrm{CVaR}_\alpha(1 - S_z) = \frac{1}{k}\sum_{j=1}^{k} F_{(j)} \quad\text{where } k = \lceil |\mathcal{Z}| \cdot \alpha \rceil$$

and $F_{(1)} \ge F_{(2)} \ge \dots$ are sorted failure rates. The constraint requires this tail failure rate to be $\le \epsilon$ (default: 0.3). An alternative mode uses pure **worst-case**: $\max_z (1 - S_z)$.

#### (c) Calibration loss (Brier score)

$$\mathcal{L}_{\text{Brier}} = \frac{1}{N}\sum_i (p_i - y_i^{\text{route}})^2$$

Where $y_i^{\text{route}}$ is the ground-truth label for "should the router have fallen back here?". This ensures the probabilistic output is well-calibrated, not just discriminative.

#### (d) Lagrangian / dual loss

The Lagrange multiplier $\lambda$ is **learned via dual ascent**:
- Router params $\psi$ **minimize** the total loss.
- $\lambda$ **maximizes** $\lambda(\mathrm{CVaR}_\alpha(1-S_z) - \epsilon)$, i.e., it pushes $\lambda$ up when the constraint is violated.

In practice the dual loss for $\lambda$ is:

$$\mathcal{L}_{\text{dual}} = -\lambda \cdot \bigl(\mathrm{CVaR}_\alpha(1-S_z) - \epsilon\bigr)$$

(negated because the $\lambda$-optimizer minimizes). The `constraint_violation` is detached so gradients of the constraint don't flow back through the router in the dual step.

### 4. Input Features (Data)

Features are generated by generate_router_features.py — a GPU-intensive pipeline that runs the SLM policy and verifier over every step of every trajectory. The resulting **13-dimensional** feature vector per decision point consists of:

| Dim | Feature | Source |
|-----|---------|--------|
| 1 | **SLM entropy** $H(\pi_\theta(\cdot \mid x_t))$ | Next-token distribution entropy from SLM forward pass |
| 2 | **Verifier score spread** (max − min over $K$ candidates) | Verifier scores $K$ SLM-generated candidate actions |
| 3 | **Mean verifier score** | Mean of $V_\phi(x_t, a^{(k)})$ |
| 4 | **Std of verifier scores** | Standard deviation over $K$ candidates |
| 5 | **Best verifier score** | $\max_k V_\phi(x_t, a^{(k)})$ |
| 6 | **Horizon fraction** $t / T$ | Normalized step progress |
| 7 | **Absolute step number** | Raw step index |
| 8 | **Context length** (normalized) | `len(context) / 10000` |
| 9–13 | **Perturbation type one-hot** | 5-dim: `[none, tool_flakiness, partial_observability, prompt_injection, distractors]` |

These capture three key uncertainty signals: **verifier confidence** (dims 2–5), **policy uncertainty** (dim 1), and **episode context** (dims 6–8, 9–13).

**Labels**: each training sample carries:
- `slm_success` (0/1): did the SLM-only episode succeed?
- `cost`: fraction of steps where the LLM teacher was used.
- `perturbation_seed`: which noise seed was applied (for CVaR grouping).

The route label $y^{\text{route}}$ is derived from risk: $y^{\text{route}} = \mathbb{1}[\rho(x_t) > \tau]$ where $\rho(x_t) = f(1 - V_\phi(x_t, \hat{a}^{\text{SLM}}),\; H(\pi_\theta(\cdot|x_t)))$.

### 5. Training Procedure

Implemented in router_trainer.py:

1. **Two separate optimizers** (primal-dual):
   - **Primal** (AdamW, cosine LR schedule): updates all router MLP weights + temperature.
   - **Dual** (Adam, fixed LR): updates only $\log\lambda$.

2. **Per batch**:
   - Forward: compute $p = r_\psi(\text{features})$.
   - Compute all four loss components.
   - **Primal step**: backprop `total_loss` through router params, clip gradients (max norm 1.0).
   - **Dual step**: backprop `dual_loss` through $\log\lambda$ only (constraint violation is detached).

3. **Hyperparameters** (defaults from code + configs):
   - Epochs: 20–50, batch size: 64–256
   - LR: 1e-3 (primal), 1e-2 (dual)
   - CVaR $\alpha$: 0.2–0.3, $\epsilon$: 0.3
   - Weight decay: 1e-4

4. **Evaluation** every 5 epochs on held-out split: Brier score, ECE, accuracy. Best checkpoint saved by Brier.

5. **Post-hoc temperature scaling**: after training, a `TemperatureScaling` module fits a single scalar $T^*$ on the validation set by minimizing NLL via bounded scalar optimization (`scipy.optimize.minimize_scalar`). This $T^*$ is written back into the router's temperature parameter, improving calibration without retraining.

---

### 6. Inference

At agent runtime (pseudocode from proposal.md):

```
1. SLM generates K candidate actions
2. Verifier scores each candidate → pick best â_SLM
3. Compute risk ρ(x_t) from verifier scores + SLM entropy
4. Router r_ψ(features) → fallback probability p
5. If p > threshold AND llm_budget > 0:
      use LLM teacher action, decrement budget
   Else:
      use â_SLM
6. Optional: self-correction loop (re-sample + verify until V ≥ accept_thresh)
```

Resource constraints are tracked by `InferenceBudget` (budget.py): max LLM calls per episode, step limit, accept threshold, and token cost tracking. This ensures the router's decisions respect a hard cost cap even if the learned policy wants to fall back frequently.

---

### 7. Key Theoretical Properties

- **Why CVaR, not average?** Average success hides catastrophic failures on adversarial perturbation seeds. CVaR focuses learning pressure on the **tail** (worst $\alpha$-fraction of seeds), directly targeting the failure modes introduced by tool noise, prompt injection, etc.
- **Why Lagrangian, not a fixed tradeoff weight?** The multiplier $\lambda$ is learned, so the system automatically adjusts how much it weighs robustness vs cost as training progresses — no manual tuning of a tradeoff hyperparameter.
- **Why calibration?** A well-calibrated router means $p=0.7$ actually corresponds to a 70% chance the SLM will fail. This is critical for the budget system: the agent can make cost-optimal decisions under uncertainty rather than relying on uncalibrated logits.
